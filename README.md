## 💡 Why this exists

> [!TIP]
> A collaborative, polyglot repository of Data Structures and Algorithms (DSA) implementations designed for pedagogical advancement and high-quality open-source contribution.

**Background:** The project serves as a comprehensive resource for high-quality algorithmic solutions across 15+ programming environments, optimized for high-velocity development cycles like Hacktoberfest.

## ✨ Features

- **Automated PR Validation**: Uses GitHub Actions to enforce documentation, complexity analysis, example usage, and syntax checks before auto-merging PRs.
- **Complexity-First Documentation**: Standardized requirement for all contributions to include Big O time and space complexity analysis.
- **Polyglot Library**: Unified directory structure and idiomatic implementation for DSA across 15+ modern programming languages.
- **Hacktoberfest Automation**: Streamlined workflow that automatically labels issues and PRs with 'Hacktoberfest' and 'good first issue'.

## 🚀 Quick Start

> [!IMPORTANT]
> Ensure you have the requisite compilers and language runtimes installed before executing localized polyglot scripts.

### Prerequisites

| Requirement | Version | Notes |
|-------------|---------|-------|
| Git | Latest | Version control system |
| C++ Compiler | C++17+ | Required for `g++` compilation |
| GHC | Latest | Required for Haskell execution |
| Node.js / NPM | 14.x+ | Required for JavaScript/TypeScript build chains |
| PHP CLI | 8.0+ | Required for PHP local testing |

> [!WARNING]
> This is a monorepo spanning 15+ environments. Dependency resolution is localized to specific language directories rather than a globally unified configuration.

### Clone & Setup

```bash
git clone https://github.com/D4rk-Pho3nix/OpenSauce.git
cd OpenSauce
```

### Build

```bash
g++ filename.cpp -o output
ghc filename.hs
npm run build
```

### Run

```bash
./output
php -S localhost:8000
ruby filename.rb
npm start
```

> [!NOTE] 
> Each directory acts as an isolated micro-environment. Navigate into specific language folders before executing build commands.

> [!TIP] 
> Reference the `CONTRIBUTING.md` file for standardized execution paradigms and testing schemas.

### Uninstall

```bash
rm -rf OpenSauce
```

## 📖 Usage

Navigate to the directory of your preferred language, compile or run the standalone algorithm files. Each file contains a main function or example usage block demonstrating the implementation.

**Examples:**

Running the Debt Simplifier in C++:
```bash
g++ DebtSimplifier.cpp -o simplify && ./simplify
```

Testing PHP Challenges locally:
```bash
cd PHP/test_karunia && php -S localhost:8000
```
